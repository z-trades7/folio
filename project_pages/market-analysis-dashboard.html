Market Analysis Dashboard

Overview

As a student and beginner trader, I faced significant challenges accessing large intraday datasets for analysis and learning. Most providers charge substantial fees or severely limit free-tier access, making it difficult to build meaningful trading skills. This inspired me to create a production-grade automated data warehouse that continuously collects 5-minute interval OHLCV data for major stocks and ETFs, making quality intraday data freely accessible to others facing the same barriers.

The system leverages the Polygon.io API to autonomously collect data for 8 major symbols (SPY, QQQ, AAPL, MSFT, GOOGL, TSLA, NVDA, AMZN) and stores it in an optimized SQLite database. A background scheduler runs daily at 5:30 PM ET, intelligently fetching only new records since the last collection while automatically skipping weekends. Starting with 60 days of historical data, the dataset grows continuously and will exceed 400 days in its first year. Users can explore the 63,232+ records through an interactive web viewer with flexible time-range filtering and CSV export capabilities.

Technical Implementation

The backend is built with Python, utilizing the requests library for API communication, pandas for data manipulation, and SQLite for storage with indexed timestamps and UNIQUE constraints to prevent duplicates. The automation layer uses the schedule library for task scheduling and runs as a background process that requires zero manual intervention after initial setup. The system detects existing data and only fetches missing records, automatically adjusting date ranges for incremental updates while respecting API rate limits to ensure continuous operation.

The frontend consists of a responsive HTML5/CSS3/JavaScript web viewer that dynamically loads data from JSON exports, allowing users to filter by symbol and select time ranges from one day to one year or view all available data. The data pipeline flows seamlessly from API through the incremental collector into SQLite, then exports to JSON for web consumption. Error handling, logging, and intelligent retry logic ensure robust operation under real-world conditions.
